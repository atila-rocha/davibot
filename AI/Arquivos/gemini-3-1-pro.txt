```elixir
defp extract_kick_args(content) do
  # Divide a string por espaços
  # Ex: ["!kick", "&lt;@123456789", "taoff"]
  args = String.split(content, ~r/\s+/, trim: true)

  case args do
    [_comando, mention | rest] -
      # Tenta capturar o ID da menção no SEGUNDO elemento da lista
      case Regex.run(~r/&lt;@!?(\d+)/, mention) do
        [_, user_id_str] -
          reason = if rest == [], do: "Motivo não especificado", else: Enum.join(rest, " ")
          {:ok, String.to_integer(user_id_str), reason}

        _ -
          {:error, "O primeiro argumento após o comando deve ser uma menção válida (ex: &lt;@user)."}
      end

    _ -
      {:error, "Uso correto: `!kick &lt;@usuario `"}
  end
end
```

### Por que isso resolve o problema?

1.  **Ignora o Comando:** Na estrutura `[_comando, mention | rest]`, o Elixir descarta o primeiro item (o `!kick`) e foca no que realmente importa: a menção e o motivo.
2.  **Regex Flexível:** A regex `~r/&lt;@!?(\d+)/` com o `!?` garante que o bot entenda a menção mesmo se o usuário tiver um apelido no servidor (o que adiciona um `!` no código da menção).
3.  **Tratamento de Espaços:** O `trim: true` no `String.split` evita que espaços duplos gerem elementos vazios na lista, o que costuma quebrar comandos de chat.

No Elixir, toda função retorna o valor da sua última expressão. No seu código:

Geralmente, as funções da `Nostrum.Api` retornam tuplas como `{:ok, %Nostrum.Struct.Message{}}` ou `{:error, reason}`. Portanto, sua função `avatar` retornará uma dessas tuplas.

### 2. O Retorno "Funcional" (Discord)
Para o usuário no Discord, a função "retorna" uma mensagem visual (o Embed).

No Elixir, o comportamento do caminho do arquivo depende de como você inicia a aplicação, mas existem regras claras para garantir que o seu bot encontre o JSON corretamente:

### 1. O Caminho Relativo
Se você definir `@xp_file "./store/xp_data.json"`, o Elixir tentará encontrar a pasta `store` a partir da **raiz do seu projeto** (onde está o arquivo `mix.exs`). 

*   **Vantagem:** É simples e direto.
*   **Risco:** Se você rodar o bot de dentro de uma subpasta ou via script diferente, o caminho relativo pode quebrar.

### 2. A Convenção da Pasta `priv`
A sua definição original (`priv/repo/xp_data.json`) é a mais recomendada no ecossistema Elixir. A pasta `priv` é o local padrão para arquivos que não são código-fonte, mas que a aplicação precisa para rodar.

Para tornar esse caminho **absoluto e seguro**, você deve usar a função `:code.priv_dir/1`. Isso garante que o bot encontre o arquivo mesmo após ser compilado ou movido.

### Como configurar o Módulo Store de forma robusta:

```elixir
defmodule Davibot.Store do
  # Busca o caminho absoluto da pasta priv do seu app :davibot
  # Se o seu app no mix.exs tiver outro nome, troque :davibot pelo nome correto
  @priv_dir :code.priv_dir(:davibot)
  @xp_file Path.join([@priv_dir, "repo", "xp_data.json"])

  def load_xp do
    if File.exists?(@xp_file) do
      @xp_file | File.read!() | Jason.decode!()
    else
      %{}
    end
  end

  def save_xp(data) do
    # Garante que a pasta priv/repo existe
    File.mkdir_p!(Path.dirname(@xp_file))
    
    json_content = Jason.encode!(data, pretty: true)
    File.write!(@xp_file, json_content)
  end
end
```

### Resumo da Recomendação:
1.  **Use `priv/`**: É o local correto para dados locais.
2.  **Evite `./`**: Caminhos que começam com ponto dependem do diretório onde o terminal está aberto, o que causa erros em produção.
3.  **Use `Path.join`**: Ele lida com as barras `/` ou `\` automaticamente, seja no Windows ou Linux.

Se você preferir manter a pasta `store/` na raiz do projeto (fora da `priv`), use `Path.expand("store/xp_data.json")` para converter o caminho relativo em absoluto no momento em que o bot inicia.

  # Substitua pela sua chave real ou use uma variável de ambiente
  @omdb_api_key "SUA_CHAVE_AQUI" 

  @doc """
  Busca dados brutos de um filme na API OMDB.

  def fetch_movie_data(movie_name) do
    # Codifica o nome do filme para ser usado na URL (ex: "The Matrix" - "The%20Matrix")
    query_name = URI.encode(movie_name)
    url = "http://www.omdbapi.com/?apikey=#{@omdb_api_key}&t=#{query_name}"

    case HTTPoison.get(url) do
      {:ok, %HTTPoison.Response{status_code: 200, body: body}} -
        case Jason.decode(body) do
          {:ok, %{"Response" = "True"} = data} -
            {:ok, data}

          {:ok, %{"Response" = "False", "Error" = reason}} -
            {:error, reason}

          {:error, _} -

      {:ok, %HTTPoison.Response{status_code: code}} -
        {:error, "Erro na API (Status: #{code})"}

      {:error, %HTTPoison.Error{reason: reason}} -
        {:error, "Falha na conexão: #{inspect(reason)}"}
